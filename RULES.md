# Distilled — 工作规则

## 一、目标

每天自动抓取指定 YouTube 频道的最新视频，提取字幕，生成一篇可以当杂志文章读的中文精华稿，推送到邮箱并保存到本地。

---

## 二、输出文件

### 格式

| 格式 | 用途 | 存放位置 |
|------|------|----------|
| PDF | 本地阅读、打印、归档 | 本地 Distilled 文件夹 |
| Word (.docx) | 内容洞察与传播素材 | 本地 Distilled 文件夹 |
| EPUB | 邮件推送，可导入电子书 App 阅读 | 仅作为邮件附件，不存本地 |

### 本地存放位置
```
~/Desktop/Distilled/
```
所有文件**直接平铺**在此目录下，不建子文件夹。

### 文件命名规则
```
{中文一句话概括内容}_{YYYY-MM-DD}.pdf
{中文一句话概括内容}_{YYYY-MM-DD}.docx
```

- 中文概括：由 AI 根据视频内容生成，要能让人一眼知道这篇文章讲什么
- 日期：抓取当天日期
- 示例：`不会写代码的人如何成为AI时代最值钱的构建者_2026-02-27.pdf`

## 三、文章写作规范

### 语言
中文写作，不是翻译腔，前提是从一开始就会在中文杂志上发表。

### 阅读时长
约 5 分钟，根据原视频内容密度灵活调整，不设硬性字数限制。

### 写作步骤

**第一步：写有力的引言**
- 采访者是谁，为什么现在值得听这个人说话
- 为什么这个话题现在重要，有什么利害关系
- 让人觉得"不读完会亏"的开头，像电影预告片

**第二步：解释"前提概念"**
- 通读全文，识别反复出现的术语和关键概念
- 用普通人能懂的语言解释，配合类比和日常例子
- 当概念第一次出现时，自然地融入行文中
- 目标读者：每天用 ChatGPT 但从未训练过模型的文科生

**第三步：只提取"宝石级洞见"**

提取标准（以下任一）：
- 真正出乎意料或反直觉
- 新颖视角，不是泛泛而谈
- 能说明大观点的具体片段或例子
- 真实感受和诚实的自我披露
- 具体策略和框架，不是抽象理论
- 影响超出专业领域，能改变日常工作和生活方式

不提取：
- 常规励志话语
- 广为人知的观点
- 表面评论和闲聊填充
- 只有从业者才懂的过度技术细节

**第四步：润色引用**
- 去掉"呃""那个""就是"等口语停顿词
- 纠正明显的转录错误和口语语法障碍
- 合并同一观点的重复表述
- 保留说话者的个性和语气

**第五步：创建连接段落**
- 宝石之间用简短的桥接段过渡
- 说明当前在讨论什么、为什么重要、如何引出下一个话题
- 把技术内容翻译成任何人都能理解的语言

**第六步：编成完整文章**
- 不是条目列表，是一篇完整的叙事文章
- 无小标题、无章节分隔（水平分隔线除外）
- 给出一个杂志风格的标题
- 开头有一段斜体引言，概括文章内容和值得读的理由
- 从头到尾不间断，自然流畅
- 不了解这个播客的人也能读懂

### 质量自查
发布前确认：
1. 读完能获得一小时视频 80% 的价值？
2. 开头是否交代了采访者和重要性？
3. 所有引用是否自然流畅？
4. 没有技术背景的读者不会迷失？
5. 前提概念的解释是否帮助理解而非制造困惑？

## 四、排版规范

### PDF / EPUB 共用样式
- 背景色：米白 `#f7f4ef`，正文区白底
- 正文字体：衬线体（Noto Serif SC / Georgia）
- 字号：17px，行高 1.95
- 最大宽度：720px，居中
- 引言段：左侧金色边框 `#c8a96e`，斜体
- 分节分隔：居中金色符号
- Meta 栏：包含来源、采访嘉宾、日期、**原视频链接**

### 原视频链接（必须包含）
每篇文章的 meta 栏必须包含原视频的可点击链接，样式为金色下划线。

## 五、自动化系统配置

### 抓取规则
- **频道**：Lenny's Podcast / Lex Fridman / Y Combinator / Andrej Karpathy / Every
- **关键词**：AI tools、vibe coding、startup、founder、product management
- **每日条数**：每天 1 条，从所有频道的最新视频中选最优（优先匹配关键词，其次看频道权重）
- **时长过滤**：20 分钟以下的视频跳过
- **推送时间**：每天早上 8:00（北京时间）
- **收件邮箱**：fxy2311@163.com

### 邮件内容
- 附件 1：EPUB 文件（可导入 Apple Books / Kindle 等电子书 App）
- 附件 2：Word 文档（内容洞察与传播素材）
- 正文：文章标题 + 简要摘要 + 原视频链接

### 技术工具

| 工具 | 用途 |
|------|------|
| `youtube-transcript-api` | 提取 YouTube 字幕 |
| `urllib` | 抓取视频标题和时长 |
| YouTube Data API v3 | 查询频道最新视频 |
| `playwright` (Chromium) | HTML → PDF |
| `ebooklib` | 生成 EPUB |
| `python-docx` | 生成 Word 文档 |
| Claude API | 生成文章、中文文件名、传播素材 |
| Gmail SMTP | 发送每日邮件 |
| GitHub Actions | 定时任务（每天 0:00 UTC = 北京时间 8:00）|

## 六、Word 文档（内容洞察与传播素材库）

每篇文章同步生成一份 `.docx` 文件，文件名与 HTML/PDF 保持一致。

内容包含五个板块（后续会调整）：
1. **最容易引爆传播的金句** — 精炼金句 + 每条的传播逻辑分析
2. **最有共鸣的故事与场景** — 具体故事片段 + 共鸣原因分析
3. **最适合展开写的洞见** — 深度内容方向 + 开场钩子句
4. **可直接套用的标题模板** — 按争议性 / 知识型 / 情感型 / 实用型分类
5. **一句话精华** — 25 字以内，适合做图片卡片或短视频封面
